
# Content structure plan (based on available draft content)

#### 0. Welcome (existing content is there) `catalogue-managers-guide/welcome.md`

#### 2. Get started
**Source:** New content needed
- **Sign in** - Access the GDI Dataset Catalogue with your credentials
- **Dashboard overview** - Basic UI orientation and key areas overview

#### 2. Manage datasets
**Source:** Test UI
- **Add datasets** - Create a collection of data resources (such as files), together with a description and other information, at a fixed URL. Users see datasets when searching for data.
- **Edit and update datasets** - Modify existing dataset information and resources
- **Organise datasets with groups** - Create and manage collections of datasets. Use this to catalogue datasets for a particular project or team, or on a particular theme, or as a simple way to help people find and search your own published datasets.

#### 3. Manage organisations
**Source:** Test UI  
- **Add organisations** - Create, manage and publish collections of datasets. Users can have different roles within an organisation, depending on their level of authorisation to create, edit and publish.
- **Manage user roles and permissions** - Control platform-level roles (Visitor, Registered User, Sysadmin) and organisation-level roles (Member, Editor, Admin)

#### 4. Set up harvesters
**Sources:** `DRAFT-CM/ckan/harvester/` folder and Test UI
- **Add harvester sources** - Import remote metadata into this catalogue. Configure remote sources such as other catalogues, CKAN instances, CSW servers or Web Accessible Folders (WAF)
- **Configure DCAT-AP harvesting** (`harvesting-dcat-ap/_index.md`) - Set up and configure step-by-step
- **Configure Fair Data Points harvesting** (`harvesting-fair-data-points/_index.md`) - Implement specialised harvesting

#### 5. Manage schemas
**Sources:** `DRAFT-CM/ckan/schemas-format-extension-and-management/` and `mapping-new-metadata-fields-strategy/`
- **Define schema format and fields** - Define JSON/YAML schema definitions
- **Update schemas** - Update schemas at runtime
- **Configure multi-schema setup** - Manage multiple dataset types

#### 6. Monitor system activity
**Source:** `DRAFT-CM/ckan/auditing-user-activity-in-ckan/_index.md`
- **Track activity streams** - Track dataset and organisation changes
- **Monitor user activity** - Monitor admin oversight capabilities
