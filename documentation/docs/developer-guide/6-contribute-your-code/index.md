Contribute your changes to GDI following best practices for collaboration and code quality.

## Contribution workflow

1. **Follow Git workflow**: Create feature branches, commit with clear messages
2. **Request code review**: Submit pull requests for team review
3. **Update documentation**: Keep docs synchronised with code changes
4. **Publish releases**: Tag versions and update changelogs

## Before you contribute

- Ensure all tests pass
- Follow coding standards
- Update relevant documentation
- Add SPDX licence headers
