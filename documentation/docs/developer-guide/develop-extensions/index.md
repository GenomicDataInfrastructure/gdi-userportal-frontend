# Develop CKAN Extensions

***NEW CONTENT NEEDED***

Learn how to develop and test CKAN extensions for the GDI User Portal.

## What you'll learn

- How to set up a local CKAN development environment
- How to understand and work with CKAN schemas
- How to test extensions effectively
- Where to find CKAN resources and documentation

## Prerequisites

Before developing extensions, ensure you have:

- **Python 3** and pip installed
- **PostgreSQL** installed and running
- **Virtual environment** tools (venv or virtualenv)
- **Git** for version control
- Basic understanding of CKAN architecture

## Extension types in GDI

The GDI User Portal uses several extension types:

- **Schema extensions**: Modify dataset metadata fields (e.g., gdi-userportal extension)
- **Harvester extensions**: Integrate external data sources (e.g., fairdatapoint extension)
- **Authentication extensions**: Handle identity and access management
- **UI extensions**: Customise CKAN interface
